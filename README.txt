Зависимости (repository) хранятся в Container, это сделано для реализации Dependency Injection. При создании приложения, создаётся container, а вместе с ним Repository. Далее создаётся UI, а вместе с ним viewModel. При создании viewModel, при помощи Factory Pattern в конструкор TradeViewModel передаётся созданный ранее Repository из Container. Во viewModel данные извлекаются с сайта при помощи корутины, запущенной внутри viewModelScope. Данные извлекаются асинхронно и записываются в изменяемую state переменную. Последняя триггерит UI, который меняется при изменении mutableUiState.
